## 1. Architecture

```
csv2json/
├── cli.py              # Argument parsing (click), entry point
├── parser.py           # CSV parsing with encoding detection
├── transformer.py      # Row → nested JSON conversion (dot-notation)
├── writer.py           # Output formatting, streaming JSON array writer
├── schema.py           # JSON Schema inference (optional, --schema flag)
└── errors.py           # Custom exceptions with user-friendly messages
```

### Key design decisions

1. **No plugin system.** [Source: simplicity] A CLI tool with 6 modules doesn't need extensibility abstractions. If someone needs custom parsers, they fork the repo.
2. **Streaming by default.** [Source: depth] The `Transform` stream approach handles arbitrarily large files. Memory usage stays constant regardless of input size.
3. **Encoding detection with fallback chain.** [Source: breadth] Try UTF-8 → detect with `chardet` → fall back to latin-1 (never fails). Log a warning on fallback so users know their data may have issues.

## 2. Implementation Phases

### Phase 1: Core pipeline (week 1)
- CSV reader with configurable delimiter and quoting
- Dot-notation header expansion (`address.city` → `{"address": {"city": ...}}`)
- Stdout JSON output with `--pretty` flag
- Basic error handling: malformed rows logged to stderr, processing continues

### Phase 2: Streaming and large files (week 1-2)
- Replace naive `csv.DictReader` with chunked streaming parser
- JSON array streaming writer (outputs valid JSON incrementally)
- Memory budget: <50MB RSS for any input size
- Backpressure: pause reading when write buffer exceeds threshold

### Phase 3: Schema inference and polish (week 2)
- `--schema` flag: scan first N rows, infer JSON Schema types
- `--output FILE` flag for file output with atomic write (temp + rename)
- `--encoding` flag to force specific encoding
- Packaging: `pyproject.toml` with `csv2json` entry point

## 3. Risk Assessment

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| Encoding detection wrong for mixed-encoding files | Medium | Data corruption | Fallback chain + `--encoding` override + stderr warning |
| Streaming parser fails on pathological CSV | Low | Crash on large files | Fuzz testing with `hypothesis`; 100MB+ test fixtures |
| Dot-notation conflicts (`a.b` and `a.b.c` both present) | Medium | Silent data loss | Detect conflicts during header parsing, fail fast with clear error |
| `--schema` inference wrong for sparse columns | High | Misleading schema | Sample first 1000 rows, flag low-confidence types as `oneOf` |
