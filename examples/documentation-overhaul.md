# Task: Plan a documentation system overhaul

Design a plan to replace an ad-hoc collection of Confluence pages, Google Docs, and README files with a unified, maintainable documentation system for a 40-person engineering team.

## Current state

- ~200 Confluence pages (mix of outdated and current, no clear ownership)
- Per-repo READMEs of varying quality
- Onboarding guides in Google Docs, partially duplicated
- API docs auto-generated but not integrated with conceptual docs
- No search across all documentation sources

## Requirements

- Docs-as-code workflow (version controlled, PR-reviewed)
- Unified search across all documentation
- Clear information architecture: tutorials, how-tos, reference, explanation
- Migration path from existing sources (not a big-bang rewrite)
- Contributor-friendly workflow that engineers will actually use

## Include in your plan

- Information architecture: top-level categories and navigation
- Tooling evaluation: static site generators, hosting, search
- Migration strategy: what to migrate first, what to archive, what to rewrite
- Ownership model: who maintains what, review process
- Quality metrics: how to measure if docs are improving
- Adoption strategy: how to get 40 engineers to contribute
- Timeline with quick wins and longer-term improvements
